## 1. 現時点で納得済みの方向性

サイト全体の方向性は以下で確定寄り。

```txt
重厚だけど読みやすい
教本っぽいけど古臭すぎない
ゼドミアは装飾ではなく理解補助
章内目次を強くする
本文を広く取る
学習の邪魔にならないが雰囲気はある
```

講義ページは「集中して読める教材」として整える。  
ホーム画面は例外的に、息抜きページ・入口ページとして装飾量を増やしてよい。

---

## 2. デザインの基本思想

正式な思想は以下。

```txt
教本の重厚さと、現代サイトの読みやすさを両立する。
ゼドミアは世界観の装飾ではなく、理解を補助する教材装置として使う。
```

デザインの仮コンセプト：

```txt
静かな私塾の教本
重厚な学習アーカイブ
黒赤の講義室
```

ただし、ユーザーから「重厚だけど読みやすい静かな私塾の学習塾」とまで言わなくてよい、ホームはシンプルな導線も大事、という指摘あり。  
したがって、言葉としては過剰に世界観を押し出さず、実際の見た目と配置で雰囲気を出す。

---

## 3. ホーム画面の方針

ホーム画面は、講義ページよりも凝ってよい。  
息抜きページ・入口ページとして、サイトの雰囲気を一番強く出す場所にする。

ただし、ホームは数学専用の入口ではない。  
数学が主なわけではないため、数学章への強い誘導ボタンは削除する。

### ホームに置くもの

```txt
・サイトタイトル
・簡潔な説明
・更新新情報
・追加ページ誘導
・日付つきのお知らせ
・管理者日誌への導線
・教科・科目選択
・下部にゼドとミアの簡単な人物紹介
```

### ホームに置かないもの

```txt
・「数学第何章を確認」など数学偏重の大きなボタン
・今回の確認ポイント一覧
・講義ページのような実用一辺倒のUI
・変な位置に来る大きなイラスト
```

ホーム上部に大きく配置したゼドミアイラストは、位置が不自然になったため削除。  
人物紹介欄やコラム内など、意味のある位置のイラストは残してよい。

---

## 4. ホームの役割

ホームは以下の役割を持つ。

```txt
1. サイト全体の入口
2. 更新情報を知らせる場所
3. 管理者日誌への入口
4. 教科・科目を選ぶ場所
5. ゼド先生とミアを軽く紹介する場所
6. 学習の合間に見ても楽しい息抜きページ
```

ホームでは学習本文そのものを詰め込みすぎない。  
訪問者が「今日は何が増えたか」「どこから学ぶか」を直感的に選べるようにする。

---

## 5. 管理者日誌の方針

管理者日誌は、更新履歴・追加ページ・編集した内容を知らせる日記のようなページにする。

今後かなり増える想定なので、年・月ごとに見返せる構造にする。

### 日誌に必要な構造

```txt
・年別アーカイブ
・月別アーカイブ
・日付つき更新情報
・追加ページへのリンク
・修正したページへのリンク
・制作メモ
```

カレンダー風、または年・月で絞り込めるアーカイブにする。  
単なる長い縦リストにすると、後から見返しづらいので避ける。

### 日誌の例

```txt
2026年6月26日
数学I 第1章「数と式の入口」を更新しました。
練習問題の解答欄を折りたたみ式にし、確認問題を追加しました。
```

## 6. 教科・科目・分野ページの方針

ホームからはまず教科・科目を選択できるようにする。

教科ページに入ると、分野ごとに選択できるページを置く。

例：

```txt
ホーム
↓
教科一覧
↓
数学
↓
数学I / 数学A / 数学II / 数学B
↓
数学Iの目次ページ
↓
章・講義一覧
↓
章ページ
```

---

## 7. 分野内ページは「本の目次」風にする

分野内ページ、たとえば「数学I」ページは、本の目次のようにする。

章だけを並べるのではなく、章の下にもう一段階小さい目次を置く。

### 目次構造

```txt
数学I
├─ 第1章　数と式の入口
│  ├─ 001 文字式と同類項
│  ├─ 002 分配法則とマイナス
│  ├─ 003 単項式を含む展開
│  ├─ 004 二つの括弧の展開
│  ├─ 005 共通因数でくくる
│  └─ 006 因数分解の考え方
│
├─ 第2章　方程式と不等式
│  ├─ 001 一次方程式
│  ├─ 002 不等式の基本
│  └─ ...
```

各小目次から、章ページ内の該当講義へ直接リンクできるようにする。

例：

```txt
001_chapter-numbers-and-expressions.html#lesson-001
001_chapter-numbers-and-expressions.html#lesson-002
```

## 8. 章ページの方針

章ページは、講義本文を読むためのページ。  
ホームほど装飾しない。  
右側の固定レールは置かず、本文を広く取る。

### 章ページに置くもの

```txt
・パンくずリスト
・章タイトル
・章の説明
・章内講義目次
・講義切り替え
・選択中の講義本文
・練習問題
・確認問題
・隠し解答
・章末まとめ
・参考文献・確認資料
・前後の章へのリンク
```

### 章ページに置かないもの

```txt
・右側固定のゼド先生コメント欄
・右側の復習カード
・右側の進捗情報
・講義と関係ない装飾的カード
・未実装の進捗表示
・押しても意味がないボタン
```

---

## 9. 章内講義目次の方針

章内講義目次は強くする。

単なるボタン列ではなく、目次として読めるカードにする。

### 講義カードに入れる情報

```txt
・講義番号
・講義タイトル
・この講義で扱う内容
・主要公式・定義
・目安時間
・難度
```

例：

```txt
001
文字式と同類項
項・係数・定数項を見分ける
公式：同類項の整理
目安：30分 / 難度：基礎
```

PCではカード型。  
スマホでは縦リスト型。  
押しやすく、現在開いている講義が分かるようにする。

## 10. 講義本文の表示方針

講義本文は、学習に集中できるようにする。

```txt
本文は広く
余計なサイド情報は消す
見出しで流れを明確にする
公式・例題・注意・練習・確認問題の見た目を分ける
```

基本構造：

```txt
001 今日の問い
002 公式・定義
003 なぜそうなるのか
004 ゼド先生の解説
005 ミアのつまずき
006 例題
007 途中式つき解説
008 間違えやすい点
009 練習問題
010 確認問題
011 解答と途中式
012 コラム
013 参考文献
```

## 11. 練習問題と解答欄の方針

練習問題の正解欄は、最初から見せない。  
ボタンを押したら表示される形式にする。

### 理由

```txt
・すぐ答えが見えると考える前に読んでしまう
・自分で解く余地を残す
・解き終わってから確認できる
```

### 表示仕様

```txt
練習問題
↓
答えを確認ボタン
↓
正解・途中式・解説が表示される
```

答えには、最終解答だけでなく途中式も必ず書く。

---

## 12. 確認問題の方針

講義の一番下に、練習問題とは別で「確認問題」を掲載する。

確認問題は、練習問題よりはるかに多くする。

ただし、確認問題は任意で解けるようにする。  
最初からすべて表示するとページが重く見えるため、ボタンを押して初めて表示される形式にする。

### 表示仕様

```txt
確認問題を開く
↓
問題一覧が表示される
↓
答えを表示
↓
途中式つきの解答が表示される
```

確認問題の答えも最初は隠す。  
答えには途中式・解法の筋道を入れる。

### 目的

```txt
・もっと解きたい人用
・復習用
・印刷して問題集化しやすくする
・練習問題だけでは足りない量を補う
```

## 13. コラムの方針

コラムは、講義始めの会話劇ではなく、キャラクター一人が豆知識や背景知識を語る形式にする。

### 基本形式

```txt
キャラクターのイラスト
＋
吹き出し
＋
豆知識・背景説明
```

### コラムの語り方

```txt
・一人が説明する
・会話ではない
・豆知識風
・説明風
・わかりやすく的確
・学習内容の背景や意味を補う
```

### 使うキャラクター

内容に合わせて、ゼド先生かミアのどちらか一人を出す。

ゼド先生向き：

```txt
・公式の意味
・歴史的背景
・抽象概念の整理
・ミスへの鋭い指摘
・学問的な補足
```

ミア向き：

```txt
・素朴な疑問
・学習者目線の気づき
・身近な例
・つまずきやすい場所の補足
```

コラムは装飾ではなく、学習の記憶に残すために使う。

## 14. ゼドミアの使い方

ゼドミアは装飾ではなく理解補助。

### 出してよい場所

```txt
・公式の意味を一言で刺すとき
・ミアが自然につまずく場所
・よくあるミスを叱る場所
・抽象的な説明を噛み砕く場所
・コラムで背景知識を説明するとき
・章末のまとめ
```

### 出しすぎない場所

```txt
・毎セクション冒頭
・ただの飾りコメント
・本文と関係ない雑談
・右側固定欄
・講義の流れを止める長い会話
```

ゼド先生は「理解の刃」。  
ミアは「つまずきの可視化」。  
この役割で使う。

## 15. 数学フォント・数式表記の方針

数学ではフォントと数式表記が大事。

`x^2` のような表記は、できるだけ使わない。  
二乗は右上に小さい数字で表示する。

HTMLでは以下のように書く。

```html
x<sup>2</sup>
```

表示例：

```txt
x²
```

今後の方針：

```txt
・x^2 ではなく x<sup>2</sup> を使う
・a^2 ではなく a<sup>2</sup> を使う
・数式は読みやすさを優先する
・正式版で数式が増えたら MathJax または KaTeX の導入も検討する
```

### MathJax / KaTeX 検討条件

以下の場合は、MathJax または KaTeX を導入してよい。

```txt
・分数が増える
・平方根が増える
・連立方程式や行列を扱う
・極限、微分積分、三角関数などが増える
・HTMLのsup/subだけでは表記が苦しくなる
```

ただし、初期段階では軽量なHTML表記を優先してよい。

## 16. ホームのイラスト方針

ホームは凝ってよいが、イラストの位置が不自然なら削る。

### 残してよいイラスト

```txt
・人物紹介欄のゼド先生
・人物紹介欄のミア
・コラム内のキャラクター
・学習導線の邪魔をしない位置の小さな挿絵
```

### 削るイラスト

```txt
・ホーム上部で変な位置に来る大きなイラスト
・導線を邪魔するイラスト
・本文やボタンと干渉するイラスト
・意味のない装飾イラスト
```

## 17. 部品デザインの方針

講義本文の部品は、機能ごとに見た目を分ける。

```txt
公式ボックス
→ 中央寄せ、薄灰背景、赤い細線。数式が目立つようにする。

例題ボックス
→ 黒い見出し、白い本文。問題と解説を区別する。

注意ボックス
→ 淡い赤背景、左に赤線。ミスしやすい点を示す。

練習問題
→ 解く場所。答えは隠す。

確認問題
→ 任意で開く大量演習。問題も答えも折りたたむ。

コラム
→ キャラ一人＋イラスト＋吹き出し。豆知識・背景説明。

参考文献
→ ページ下部に静かに配置。主張しすぎない。
```

```txt
ホーム：
凝ってよい。息抜きページ。更新情報・日誌・教科選択・人物紹介を置く。

日誌：

教科ページ：
教科から分野へ進む。

分野ページ：
本の目次風。章の下に講義小目次を置く。

章ページ：
講義本文中心。右側固定欄なし。章内講義目次を強くする。

答えは隠す。押したら途中式つきで表示。

確認問題：
任意で開く。練習問題より多くする。答えも隠す。途中式つき。

キャラ一人がイラスト＋吹き出しで豆知識を説明する。

数式：
x^2 ではなく x² のような上付き表示を使う。
```

# 教えてゼド先生：追加ルール統合版

作成日：2026-06-26  
用途：今後の「教えてゼド先生」サイト制作時に参照する追加方針。
位置づけ：既存のサイト構造ルール・デザイン方針に追加する、教科順・シラバス制作・サイト上の番号表記・イラスト使用に関する統合ルール。

## 1. 教科の並び順

教科一覧では、哲学を一番最初に配置する。

```txt
001 哲学
002 数学
003 国語
004 理科
005 社会
```

フォルダ構成は以下を基本にする。

```txt
010_subjects/
├─ 001_philosophy/
├─ 002_math/
├─ 003_japanese/
├─ 004_science/
└─ 005_social-studies/
```

数学は重要教科だが、「教えてゼド先生」全体の思想・学習方針の中心には哲学を置く。ホーム、教科一覧、ナビゲーションでは哲学を最初に表示する。

## 2. 各教科はシラバス作成から始める

新しい教科を作成するときは、いきなり講義ページや章ページを作らない。必ず最初に、その教科全体のシラバスを作成する。

基本ファイル名は以下とする。

```txt
000_syllabus.html
```

例：

```txt
010_subjects/001_philosophy/000_syllabus.html
010_subjects/002_math/000_syllabus.html
010_subjects/003_japanese/000_syllabus.html
```

## 3. 教科制作の基本順序

```txt
教科構想
↓
シラバス
↓
分野
↓
章
↓
講義
↓
問題
↓
参考資料
```

1. 教科全体のシラバスを作る
2. 教科の目的・到達目標を決める
3. 分野構成を決める
4. 章構成を決める
5. 各章に入れる講義一覧を決める
6. 章ページを作る
7. 講義本文を作る
8. 練習問題・確認問題・解答を作る
9. 参考文献・確認資料を整える
10. 必要に応じて修正・増補する

## 4. シラバスに必ず入れる内容

- この教科で何を学ぶのか
- この教科を学ぶ目的
- 到達目標
- 対象レベル
- 全体カリキュラム
- 分野一覧
- 章一覧
- 推奨学習順
- 各分野の概要
- 演習・課題の方針
- 評価・確認方法
- 参考文献・確認資料

可能であれば、以下も入れる。

- この教科を学ぶ意味
- 他教科との接続
- 創作や実生活への応用
- ゼド先生からの学習方針
- ミア視点のつまずきやすい点

## 5. 授業構想の品質基準

シラバスは、単なる目次ではなく、授業全体の設計図として作る。目標は、学校の授業と同等ではなく、それを上回る品質にすること。

- 全体の流れに必然性がある
- なぜその順番で学ぶのかが分かる
- 基礎から応用へ自然に進む
- 知識の暗記で終わらない
- 考え方、使い方、応用先まで示す
- 各章の目的が明確である
- 各講義の役割が重複しすぎない
- 演習や確認問題まで見越して設計する
- 参考文献や確認資料の方針がある

## 6. 哲学教科の位置づけ

哲学は、「教えてゼド先生」の中心教科として扱う。哲学は、単に哲学史や用語を覚えることではなく、以下を育てる教科として設計する。

- 問いを立てる力
- 概念を整理する力
- 前提を疑う力
- 論理的に考える力
- 世界観や創作を深める力
- 他教科を深く理解するための土台

## 7. サイト上の番号表示ルール

フォルダ名・ファイル名・URL管理では、これまで通り三桁番号を使う。

```txt
001_philosophy
002_math
001_math1
001_chapter-numbers-and-expressions.html
#lesson-001
```

ただし、サイト上に表示される番号は三桁にしない。

採用する表記：

```txt
第1章　数と式の入口
第2章　方程式と不等式
第3章　関数

第1講　文字式と同類項
第2講　分配法則とマイナス
第3講　単項式を含む展開
```

カードの左上など、短く見せたい場所では `1`、`2`、`3` のように表示してよい。

三桁番号は以下の場所に限定する。

- フォルダ名
- ファイル名
- HTMLのid
- URLハッシュ
- JSONなどの管理データ

読者に見える場所では使わない。

- ページ見出し
- 章タイトル
- 講義カード
- パンくずリスト
- 目次
- ボタン表示
- 本文中の案内

## 8. イラスト使用ルール

ゼド先生・ミアのイラストは、用途に応じて使い分ける。

```txt
バストアップ：
吹き出し、短評、つまずき、コラム、注意、補足

立ち絵：
人物紹介、ホーム、教科導入、章導入、特別ページ、ページ装飾
```

### バストアップイラスト

主に吹き出し・会話・短い補足コメント用に使う。

- ゼド先生の一言解説
- ミアのつまずき
- コラムの吹き出し
- 注意ポイント
- 公式の意味を短く説明する場所
- 章末の短いまとめコメント

### 立ち絵イラスト

立ち絵は、吹き出し用の小アイコンとしては使わない。キャラクターの存在感を出すための大きめのビジュアルとして使う。

- ホーム下部の人物紹介
- 教科ページの導入部分
- 章ページ冒頭の案内役
- 節目のページ
- 管理者日誌の装飾
- 特別講義ページ
- コラム外側のビジュアルアクセント

## 9. 画像フォルダとCSS分類

```txt
090_assets/images/
├─ zed/
│  ├─ bust/
│  ├─ standing/
│  └─ action/
│
└─ mia/
   ├─ bust/
   ├─ standing/
   └─ action/
```

CSSでは、用途ごとにクラスを分ける。

```html
<img class="character-bust" src="..." alt="ゼド先生">
<img class="character-standing" src="..." alt="ゼド先生">
```

## 10. 今後の制作時に守ること

1. 教科一覧では哲学を一番上に置く
2. 哲学フォルダは `001_philosophy` とする
3. 数学フォルダは `002_math` とする
4. 各教科には `000_syllabus.html` を作る
5. 教科制作はシラバス作成から始める
6. シラバスでは教科全体の目的・到達目標・構成を決める
7. 章や講義は、シラバスに沿って作る
8. 学校の授業と同等ではなく、それ以上の品質を目指す
9. 目次は単なる一覧ではなく、授業構想として機能させる
10. 教科全体の流れが決まる前に講義を量産しない
11. 内部管理では `001 / 002 / 003` を使う
12. サイト上の表示では `第1章 / 第2章 / 第3章` にする
13. 講義も表示上は `第1講 / 第2講 / 第3講` にする
14. 三桁番号は読者に見える場所では出さない
15. バストアップイラストは吹き出し・補足・コラム用に使う
16. 立ち絵は吹き出し用に使わない
17. 立ち絵はホーム、人物紹介、教科導入、章導入、特別ページで使う
18. イラストは本文や導線を邪魔しない場所に置く
19. 画像フォルダやCSSでは、バストアップと立ち絵を用途別に分ける

# 教えてゼド先生：サイト制作方針・構造ルール

作成日：2026-06-25  
用途：今後の「教えてゼド先生」サイト制作時に参照する情報源用ファイル。  
目的：このチャット内で決まったサイト構造・運用方針・講義制作ルールを固定し、今後のHTML生成やページ追加時の基準にする。

## 2026-06-26 追記：教科順とシラバス先行ルール

教科一覧では、哲学を最初に置く。

```txt
001_philosophy
002_math
003_japanese
004_science
005_social-studies
```

また、新しい教科を作るときは、最初に `000_syllabus.html` を作成する。  
シラバスには、教科全体の目的、到達目標、全体カリキュラム、演習方針、講義品質基準を入れる。

## 1. 基本方針

「教えてゼド先生」は、今後かなり大規模な学習サイトとして育てる前提で制作する。

従来のような「1枚のHTMLにすべてを詰め込むプレビュー形式」ではなく、正式版では以下の方針を採用する。

- VS Codeでユーザー自身が編集・追加しやすい構造にする。
- 公開後も、同じURLのままページ内容だけ更新できるようにする。
- ページ追加・修正のたびにサイト全体を作り直すのではなく、1ページ単位・1章単位で追加できる構造にする。
- サイト全体は複数HTMLで構成する。
- 講義ページは「章ページ単位」で作り、章ページ内に複数講義を収録する。
- 講義は章ページ内でタブ・カード・ボタンなどにより切り替え、「中ページが変わったように見せる」方式にする。
- デザインはまだ確定しない。まずは構造を確定する。
- 今の赤黒白の雰囲気や教本感は参考にするが、不要なUIは削る。

---

## 2. 公開・更新方針

公開後もURLを変えずに内容だけ更新できるようにする。

例：

```txt
/010_subjects/002_math/001_math1/001_chapter-numbers-and-expressions.html
```

このファイルの中身だけを編集して再アップロードすれば、URLは変わらず内容だけ更新される。

公開先候補：

- GitHub Pages
- Cloudflare Pages
- Netlify

VS Codeで編集し、GitHubなどにアップロードして公開する運用を想定する。

---

## 3. 三桁番号ルール

サイトは将来的にかなり大きくする予定なので、フォルダ名・ファイル名の先頭には三桁番号を付ける。

目的：

- VS Code上で見たときに並び順が崩れないようにする。
- 優先度の高いページや先に読むページが上に来るようにする。
- 将来ページが増えても番号の余裕を持たせる。

例：

```txt
001_philosophy
002_math
003_japanese
004_science
005_social-studies
```

トップページだけは例外として `index.html` のままにする。

## 5. ページ階層

サイト全体の階層は以下を基本とする。

```txt

↓
管理者日誌
↓
教科一覧
↓
教科ページ
↓
分野ページ
↓
章ページ
↓
章ページ内の講義切り替え
```

具体例：

```txt
ホーム：/index.html
管理者日誌：/001_admin-log/index.html
教科一覧：/010_subjects/index.html
数学：/010_subjects/002_math/index.html
数学I：/010_subjects/002_math/001_math1/index.html
数学I 第1章：/010_subjects/002_math/001_math1/001_chapter-numbers-and-expressions.html
```

## 6. 講義ページの基本方式

講義は、1講義1HTMLではなく、1章1HTMLにまとめる。

例：

```txt
001_chapter-numbers-and-expressions.html
```

この章ページ内に、以下のように複数講義を収録する。

```txt
001 文字式と同類項
002 分配法則とマイナス
003 単項式を含む展開
004 二つの括弧の展開
005 共通因数でくくる
006 因数分解の考え方
```

画面上では、タブ・ボタン・カードなどで講義を切り替える。

実際には同じHTML内で表示を切り替えるが、見た目としては中ページが変わったように見せる。

---

## 7. 章内URLルール

章ページ内の講義は、URLの末尾にハッシュを付けて直接開けるようにする。

例：

```txt
001_chapter-numbers-and-expressions.html#lesson-001
001_chapter-numbers-and-expressions.html#lesson-002
001_chapter-numbers-and-expressions.html#lesson-003
```

この方式により、同じ章HTMLの中にある特定の講義へ直接リンクできる。

基本仕様：

```txt
章ページURL + #lesson-三桁番号
```

例：

```txt
/010_subjects/002_math/001_math1/001_chapter-numbers-and-expressions.html#lesson-003
```

## 9. 章内講義目次の方針

章内講義目次は、単なるボタン列ではなく、目次として分かりやすいデザインにする。

PCではカード型を基本とする。

各カードに入れる情報：

- 講義番号
- 講義タイトル
- この講義で扱う内容
- 主要公式・定義
- 目安時間
- 難度

例：

```txt
001
文字式と同類項
項・係数・定数項を見分ける
公式：同類項の整理
目安：30分 / 難度：基礎
```

スマホでは縦リスト型にし、押しやすくする。

## 10. 講義本文の構造

各講義本文は以下の流れを基本とする。

```txt
001 今日の問い
002 公式・定義
003 なぜそうなるのか
004 ゼド先生の解説
005 ミアのつまずき
006 例題
007 途中式つき解説
008 間違えやすい点
009 練習問題
010 ミニテスト
011 解答と採点基準
012 コラム
013 参考文献メモ
```

ただし、毎回すべてを無理に入れる必要はない。

最低限必須：

```txt
・今日の問い
・公式・定義
・なぜそうなるのか
・例題
・練習問題
・解答
・参考資料
```

ゼドミア会話やコラムは、理解を助ける場所に必要な分だけ入れる。

## 11. 講義品質ルール

講義ページは、実際の授業より分かりやすく、腑に落ちやすい教材を目指す。

制作時のルール：

- 公式は必ず明記する。
- 定義は曖昧にしない。
- 途中式を省略しすぎない。
- 「なぜそうなるのか」を入れる。
- 例題は実際に解き直して検算する。
- 間違えやすい点を最低2つ以上入れる。
- ミアがつまずく場所を意図的に入れる。
- ゼド先生の指摘は、理解に意味がある場所で使う。
- 最後に参考文献・確認資料を明記する。

数学で特に重視すること：

- 公式の意味
- どこを見る公式なのか
- 何を探す公式なのか
- なぜその形になるのか
- どこでミスしやすいのか
- 計算の検算
- 符号ミス・次数ミス・同類項ミスの明示

---

## 12. 公式・定義の扱い

公式は単に載せるだけではなく、必ず意味を説明する。

例：

```txt
(a + b)(c + d) = ac + ad + bc + bd
```

説明すべきこと：

```txt
・この公式は何をするためのものか
・どこに何を掛けているのか
・なぜこの形になるのか
・逆向きに見ると何につながるのか
・どこでミスしやすいのか
```

---

## 13. コラムミニコーナー

講義内に、たまにコラムミニコーナーを入れる。

目的：

- 公式や分野がなぜ生まれたのかを知る。
- 学習内容を歴史・思想・実生活・科学技術につなげる。
- 暗記だけでなく、知識の背景を理解する。

候補名：

```txt
ゼド先生の余談
ミアの素朴な疑問
公式の生まれた場所
歴史の寄り道
なぜこの考えが必要だったのか
この知識、どこで使うの？
```

コラム例：

```txt
COLUMN
なぜ「因数分解」という考えが生まれたのか

式を分解する目的は、ただ計算を楽にするためだけではない。
複雑に見えるものを、より単純な部品に戻すためだ。
この発想は、方程式を解くとき、関数の性質を見るとき、さらには物理やコンピュータの計算でも使われる。
```

## 14. 情報源・正誤確認ルール

講義制作時は、ちゃんとした情報源を参照して正誤確認する。

優先する情報源：

- 文部科学省などの公的資料
- 学習指導要領
- 教科書範囲に準拠した資料
- 大学・研究機関の公開資料
- 信頼できる教育機関の教材
- 論文・一次資料
- 公的機関や研究機関の図表

外部情報を使う場合：

- 参考文献・確認資料欄に明記する。
- 画像を使う場合は、使用可能なライセンスか確認する。
- ニュースは引用ではなく要約中心にする。
- 個人ブログやSNSだけを根拠にしない。
- AI生成まとめ記事だけで正誤確認しない。

外部画像の利用候補：

- Wikimedia Commonsなどライセンス確認できる画像
- NASA、国立天文台、気象庁、文部科学省などの公開資料
- 大学・研究機関・公的機関の図表

## 15. 参考文献欄

章ページまたは講義ページの最後に、参考文献・確認資料欄を入れる。

```html
    <li>必要に応じて大学・研究機関・公的機関の公開資料</li>
  </ul>
</section>
```

## 17. 添付HTMLから削る・改善する要素

既存のプレビューHTMLには、正式版では不要・未確定・見にくい部分がある。

削除または保留するもの：

- 右側のゼド先生固定コメント欄
- 右側の復習カード
- 右側の進捗情報
- 右側の固定レール全体
- 上部の進捗リング
- 未実装の進捗表示
- 未実装のクイックリンク
- 押しても機能しないPDFボタン
- 未実装の問題集、用語帳、PDF書庫ボタン
- 講義と関係ない装飾的カード
- base64で埋め込まれた画像
- HTML内に直書きされた大量のCSS

理由：

- 講義本文に集中しにくい。
- 画面が情報過多になる。
- 未実装機能があるとユーザーが混乱する。
- 右側レールが本文幅を圧迫する。
- 巨大化したときに保守しにくい。

## 18. レイアウト方針

正式版の講義ページは、3カラムではなく2カラムまでにする。

PC：

```txt
左：教科・分野ナビ
中央：章ヘッダー、章内講義目次、講義本文
右：なし
```

スマホ：

```txt
上：メニュー
下：本文
```

右側の固定ゼド先生コメント欄は廃止する。

ゼド先生のコメントは、右側に固定表示するのではなく、本文中の必要な箇所に吹き出し・短評として入れる。

## 21. HTML・CSS・JSの分離

正式版では、HTML内にCSSや画像を大量に埋め込まない。

HTML：本文構造  
CSS：デザイン  
JS：講義切り替え・URLハッシュ処理・将来の進捗保存  
画像：外部画像ファイル  
データ：JSONなど  

CSSは以下のように分ける。

```txt
090_assets/css/base.css
090_assets/css/layout.css
090_assets/css/lesson.css
090_assets/css/components.css
```

JSは以下のように分ける。

```txt
090_assets/js/main.js
090_assets/js/lesson-switcher.js
090_assets/js/progress.js
```

画像は以下のように外部ファイルとして読み込む。

```html
<img src="../../../090_assets/images/zed/zed_normal.png" alt="ゼド先生">
```

## 22. 講義ページ制作時の実行ルール

今後、講義ページを制作するときは以下の流れで作る。

```txt
1. 対象教科・分野・章を確認する
2. 章内に入れる講義一覧を決める
3. 公式・定義・範囲を情報源で確認する
4. 章ページのHTMLを作る
5. 講義切り替え用の lesson-001 形式を設定する
6. 各講義に公式・説明・例題・練習問題を入れる
7. ゼドミア会話は必要な場所だけ入れる
8. コラムを必要に応じて入れる
9. 解答・採点基準を作る
10. 参考文献・確認資料を明記する
11. スマホ表示とURLハッシュ切り替えを確認する
```

## 24. 要約

正式方針：

```txt
・三桁番号で管理する
・VS Codeで編集しやすい構造にする
・公開後もURLを固定して更新できるようにする
・サイト全体は複数HTMLに分ける
・章ページは1HTMLにする
・章ページ内に複数講義を入れる
・講義はタブ/カードで切り替える
・URLは #lesson-001 形式で変化させる
・右側のゼド先生コメント欄や進捗欄は削る
・章内講義目次を目次らしく分かりやすくする
・講義本文は公式、定義、なぜ、例題、練習、解答、参考文献を重視する
・外部情報は信頼できる情報源で確認する
・デザインは未確定。まず構造を確定する
```
